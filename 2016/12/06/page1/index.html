你可能不了解的前端系列之CSS : border-radius

2016-12-06


相信大家在开发中都会经常使用到border-radius这个属性, 然而对于border-radius: 50%; 或者

这样的border-radius是怎么计算来的是否了解呢？

border-radius属性用来定义盒模型四角的弧度, 学习能力强的同学可以参考 CSS Backgrounds and Borders Module Level 3

像我一样的 学渣 萌新请看这段~

已经了解border-radius基本原理的同学请点这里

border-radius 是 border-top-left-radius, border-top-right-radius, border-bottom-right-radius, border-bottom-left-radius四个属性的简写,用于同时设置四边的圆角。

我们先从border---radius开始。

border---radius用来设置 左上、右上、右下、左下四个角的圆角

border---radius接收两个值,分别用来设置圆角弧度的水平半径和垂直半径

两个值的类型可以是具体的长度单位也可以是百分比:

让我们举个栗子🌰 来看看发生了什么

<div style="height: 100px;width: 100px;background: pink;border-top-left-radius: 50px 25px;"></div>

那么上面的DIV就是如下的效果:

我们来图解一下它的圆角是如何计算的。

百分比的话 ,水平半径会按照BOX的宽度得到计算值, 垂直半径会按照BOX的高度会得到计算之

<div style="height: 100px;width: 100px;background: pink;border-top-left-radius: 50% 25%;"></div>

和上面的DIV会有一样的效果

border-radius接受四个值, 用来依次设置左上、右上、右下、左下四个圆角的水平半径和垂直半径, 每个角只能接受一个值,意味着使用border-radius设置圆角的话水平半径和垂直半径只能相等(深刻理解)

<div style="height: 100px;width: 100px;background: pink;border-radius: 25px 10px 15px 30px;"></div>

本篇重点从这里开始～

上面已经有了border-radius的基本使用方法介绍, 下面我们来看一些特殊的情况。

Case 1

如果我的盒子具有border, 那么border-radius该如何计算呢？

W3C爸爸标准定义,当盒子同时存在border和border-radius, border的内外弧度是分开计算的(如果不知道什么是border请放弃前端);

其内侧弧度(以下简称内弧度)的值为border-radius设置的水平半径、垂直半径减去border的宽度,当其值为负的时, 取之为0;

直男画图困难系列,就不画图了,大家看下面的例子:

给出如下例子(均采用border-sizing: border-box;)

<div style="height: 70px;width: 70px;border-top:solid 10px;border-left: solid 10px; border-bottom:solid 10px;border-right:10px solid;background: pink;border-top-left-radius: 50px 25px;"></div>

当border-left 为51px的时候, 50-51为负,水平半径为0,所以失去了弧度

<div style="height: 70px;width: 70px;border-top:solid 10px;border-left: solid 10px; border-bottom:solid 10px;border-right:10px solid;background: pink;border-top-left-radius: 50px 25px;"></div>

装逼时间到

下面这种border-radius, 相信大家会经常使用:

<div style="height: 50px;width: 100px;background: pink;border-radius: 25px;"></div>

那么现在我会告诉你, 所有在这里设置超过25px的,最终都会计算成这个图形的样子。

在W3C中对border-radius最终的计算值有一个衰减算法,具体如下:

Let f = min(Li/Si), where i ∈ {top, right, bottom, left}, Si is the sum of the two corresponding radii of the corners on side i, and Ltop = Lbottom = the width of the box, and Lleft = Lright = the height of the box. If f < 1, then all corner radii are reduced by multiplying them by f.

大致翻译一下, 就是当你的垂直半径和水平半径之和大于盒子宽高中最小的那一个值的时候，垂直半径和水平半径最后会乘以一个值, 值为宽高中最小的值除以水平半径与垂直半径之和(真的不是绕口令吗)

那么下面给出两个推论：

推论1 : 垂直半径和水平半径之和大于盒子宽高中最小的那一个值的, 且垂直半径与水平半径的比越接近1,则最终的计算值越接近垂直半径和水平半径之和大于盒子宽高中最小的那一个值的一半

证明: 设垂直半径为V, 水平半径为h0, 宽高为w、h

value(ho)　= limit(v→h0) ho*min(h, w)/ho + v
　　　　　　= ho*min(h,w)/2ho
　　　　　　= min(h, w)/2

推论2 : 垂直半径和水平半径之和大于盒子宽高中最小的那一个值的, 且垂直半径与水平半径一方为另一方的高阶无穷小,则弧度为0

value(ho)　= limit(v → ∞) ho*min(h, w)/ho + v
　　　　　　= 0

value(v)　= limit(v → ∞) v*min(h, w)/ho + v
　　　　　　= min(h,w);

所以: 弧度为0
